# Telegram бот для анонимной предложки

Бот принимает сообщения только в личке, автоматически режет запрещённый контент по правилам и отправляет допустимые заявки в чат модерации. Поддерживаются обычные текстовые сообщения, фото с подписью и видео с подписью. В сообщении для админов видны `username` автора, итог решения и кто его принял. После нажатия `Опубликовать` сообщение уходит в целевой чат анонимно, без данных автора.

## Что умеет

- анонимный приём сообщений через личку боту
- rule-based фильтр без AI и без агентов
- блокировка 18+, оскорблений, экстремистского контента
- блокировка ссылок и внешних контактов
- приём фото с подписью
- приём видео с подписью
- модерация через inline-кнопки в Telegram
- уведомление автора о публикации или отклонении
- хранение заявок в SQLite

## Быстрый старт

1. Создай бота через `@BotFather` и получи токен.
2. Создай чат для модерации и канал или чат для публикации.
3. Добавь бота в оба чата.
4. Выдай боту право отправлять сообщения.
5. Узнай `chat_id` для модерации и публикации.
6. Скопируй `.env.example` в `.env`.
7. Заполни переменные окружения.
8. Установи зависимости:

```powershell
npm.cmd install
```

9. Запусти бота в разработке:

```powershell
npm.cmd run dev
```

10. Или собери production-версию:

```powershell
npm.cmd run build
npm.cmd start
```

Чтобы быстро узнать ID:

- напиши боту `/chatid` в личке, чтобы получить свой `user id`
- добавь бота в группу или канал и отправь туда `/chatid`, чтобы получить `chat_id`

## Переменные окружения

- `BOT_TOKEN` - токен Telegram-бота
- `MODERATION_CHAT_ID` - чат для заявок на модерацию
- `TARGET_CHAT_ID` - чат или канал для публикации
- `ADMIN_USER_IDS` - список Telegram user id через запятую; если оставить пустым, модерировать сможет любой участник чата модерации
- `DISPLAY_TIMEZONE` - часовой пояс для отображения времени в админских сообщениях, например `Europe/Moscow`

## Как работает фильтр

Фильтр локальный и построен на правилах из [`src/filter.ts`](./src/filter.ts):

- регулярные выражения для 18+ лексики
- регулярные выражения для оскорблений
- регулярные выражения для нацистского и экстремистского контента
- запрет ссылок и внешних контактов
- ограничение длины сообщения

Если нужна более жёсткая фильтрация, просто дополняй список в [`data/banned-terms.txt`](./data/banned-terms.txt).

## Структура проекта

- `src/index.ts` - Telegram-бот и модерационные кнопки
- `src/filter.ts` - фильтрация запрещённого контента
- `src/storage.ts` - SQLite-хранилище заявок, включая фото и текст
- `src/config.ts` - чтение `.env`
- `data/submissions.db` - база SQLite с историей заявок
- `docs/FUNCTIONALITY.md` - подробное описание функционала проекта
